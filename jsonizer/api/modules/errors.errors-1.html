    Dynamically create a user error class with the name given.

    paramtype

    Type - Additional custom properties (optional) allowed in the error. The properties can be set after calling new.

    see

    see

    getName

    Type parameters

    • Type = unknown

    Parameters

    • name: string

      The name of the class.

    • unique: boolean = true

      if true (by default), a singleton with that name will be returned ; if false, a new class will be returned if that name was not previously used to define a singleton.

    • Optional code: string | number

      Conveniently, an error code may be bound to the class, because it is a common practice. If a code is defined, the singleton must have the same code, otherwise another class will be generated.

    Returns TypedError<Type>
